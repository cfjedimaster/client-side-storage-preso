Working with Static Sites
the JAMStack

Static Site Generators (SSG)

  • They make static sites!
  • They give you a dynamic app server locally (typically using template languages like Markdown, Pug, SASS, etc)
  • They give you a way to work with data (without a database)
  • They output to static

Options

  • https://staticsitegenerators.net
  • https://www.staticgen.com

For Today

11ty

  • https://www.11ty.io
  • Open Source
  • Node based
  • Template Languages: Nunjucks, Liquid, Handlebars, Mustache, EJS, Haml,

Installation

npm install -g @11ty/eleventy

Commands (Some)

eleventy

eleventy --input=. --output=./_site

eleventy --serve

Demo

Show 11ty_one, a single file. Run eleventy to show the output, then eleventy serve. Demonstrate lack of auto reload Show 11ty_two, example of Liquid

Layouts

  • Specified in front matter (YAML, JSON, JS)
  • Loaded from /_includes
  • Can use any template language
  • Can be chained

Front Matter


---
layout: something
title: a value
something: another value
---

Demo

11ty_three - layout. front matter, show title, show link chnaging, show content

Working with Data

  • A way to define variables for your site
  • Front matter
  • File-based data (JSON or JS)
  • For one other file, for a directory, for a site (global)
  • For global data, stored in _data
